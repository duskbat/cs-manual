# Kafka

## 架构

Producer
Broker(主从): 在一个 Broker 上不是所有的 Partition 都是主, 有可能也是从
Consumer
Zookeeper

## 消息不丢失

要保证 发送, 存储, 消费 链路不丢失

关键在于发送, 因为 Kafka 的发送是异步的, 所以建议使用重试的方法

- 当主从都确认时认为发送成功
- 重试次数
- 重试间隔

存储有持久化, 主备 和 ISR(与主节点保持同步的节点)

- 副本数量配置
- ISR 数量
- 不允许非 ISR 被选举为 leader

消费者有消费者组, 只要保证不胡乱提交 offset 即可

## 重复消费

消费者组可以让同组内的消费者不会重复消费

消费者手动提交 offset 控制消费情况

客户端幂等

## 三种消费语义

- 至少一次(消费失败不提交偏移量) 默认
- 最多一次
- 精确一次(事务)

## 顺序消费

要实现顺序消费, 将 Topic 只创建一个 partition, 或者将需要顺序消费的都发到同一个 partition
