## Java 8

from 2014

### Interface

default 方法
static 方法

虽然有了默认实现, 但跟抽象类还是有些区别的, interface 的方法由 public abstract 修饰, 属性 public static final
这种变动是为了减少接口和实现之间的强约束

### 函数式接口

有且只有一个抽象方法(可以有多个非抽象方法)的接口
跟 @FunctionalInterface 注解无关
内置的一些:

- Predicate
- Function
- Supplier
- Consumer
- Comparator

### Lambda 表达式

是一个匿名函数, java8 允许函数作为参数
也能用::关键字调用, 返回类型必须是 functional-interface

> 以前想传递函数只能用匿名内部类

参数都是 final, 不能使用 default 方法

编译后可能是个静态方法

### Stream

- 参数是函数式接口
- 一个 Sream 只能使用 1 次
- 不改变数据

延迟执行, 等返回一个非 Stream 的方法后才执行

parallelStream 使用 ForkJoin 框架 ForkJoinPool

### 多重注解

```java
@Retention(RetentionPolicy.RUNTIME)
@interface Hints {
    Hint[] value();
}

@Repeatable(Hints.class)
@interface Hint {
    String value();
}

// 旧版使用
@Hints({@Hint("hint1"), @Hint("hint2")})
class Person1 {
}

// 新版使用 而且能够通过反射获取到 @Hints 注解
@Hint("hint1")
@Hint("hint2")
class Person2 {
}
```

### Stream & Optional

- 空 stream
- takeWhile() 依次获取满足条件的元素, 遇到第一个不满足的停止
- dropWhile() 从第一个不满足的开始, 获取满足条件的元素, 第一个不满足的也算
  > 个人感觉有点复杂
- iterate()
  ```java
  // 使用原始 iterate() 方法输出数字 1~10
  Stream.iterate(1, i -> i + 1).limit(10).forEach(System.out::println);
  // 使用新的 iterate() 重载方法输出数字 1~10
  Stream.iterate(1, i -> i <= 10, i -> i + 1).forEach(System.out::println);
  ```
- ifPresentOrElse()
  方法接受两个参数 Consumer 和 Runnable，如果 Optional 不为空调用 Consumer 参数，为空则调用 Runnable 参数
- or()
  ```java
  lookup = name -> finalLookup.lookup(name).or(() -> fallbackLookup.lookup(name));
  ```
